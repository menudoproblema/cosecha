  Propuesta: Cosecha Deep Observability (v2 - The Harness)

  Esta arquitectura no solo arregla el coverage; crea un entorno donde la ejecucin del cdigo es consciente, comunicativa y adaptable en tiempo real.

  1. El Kernel: packages/cxp-supervisor (Independiente)
  Extraer toda la lgica de ejecucin a un paquete de infraestructura pura. Su nica misin: Garantizar que un proceso Python sea observado de principio a fin.

   - Mecanismo de Inyeccin: Abandonar el argv_prefix en favor de sitecustomize.py. El supervisor inyecta un archivo temporal en el PYTHONPATH del hijo.
   - Beneficio: Esto instrumenta automticamente todos los procesos hijos (workers, subprocesses directos), no solo el proceso inicial. Es la nica forma de garantizar cobertura total en sistemas distribuidos o
     multiproceso.
   - Aislamiento: Creacin del Shadow Execution Context (directorio temporal ciego al usuario) donde caen todos los archivos de datos (.coverage, .prof, .jsonl).

  2. El Sidecar: Bridge CXP en Tiempo Real
  Sustituir el archivo de metadata JSON por un Canal Sidecar Activo (Unix Socket / Named Pipe).

   - Handshake Inicial: Al arrancar, el proceso hijo (sitecustomize) se conecta al bridge y recibe la "Receta de Instrumentacin" (qu slots usar, qu filtros aplicar).
   - Stream de Eventos: Los instrumentadores emiten eventos estructurados en Sobres CXP mnimos mientras los tests corren:
       - instrumentation.heartbeat: Para detectar si el instrumentador ha muerto.
       - instrumentation.artifact_detected: Informa al padre de un nuevo archivo de datos antes de que termine el proceso.
       - instrumentation.live_event: Telemetra live (tracing/profiling) para consumo inmediato por MCP/LLM.
   - Buffering Obligatorio: El lado del hijo implementa una cola asncrona para no bloquear la ejecucin de los tests por I/O de telemetra.

  3. El Cerebro: Orquestacin Adaptativa (Git + KB)
  En lugar de instrumentar todo el monorepo, la shell de Cosecha realiza un anlisis de impacto antes de invocar al supervisor.

   - Filtro de Inteligencia: Cruza el git diff con el grafo de dependencias de la Knowledge Base.
   - Instruccin Selectiva: Enva al supervisor una lista blanca de mdulos a instrumentar.
   - Voto de Desconfianza: Si durante la ejecucin el Sidecar detecta que se est ejecutando cdigo modificado que no estaba en la lista blanca (va sys.settrace), el supervisor emite un Warning de Integridad de
     Cobertura en tiempo real.

  4. Composicin por Matriz de Capacidades
  Resolver el conflicto de slots de Python (trace, profile, monitoring) de forma declarativa.

   - Capability Matrix: Cada instrumento declara qu recursos del runtime de Python necesita.
   - Deteccin de Conflictos: El supervisor valida la compatibilidad antes de lanzar. Si el usuario pide coverage + profiling y colisionan, el supervisor inyecta un Multiplexor de Trazado (un nico hook de settrace
     que llama a ambos secuencialmente) o falla con un diagnstico claro.
